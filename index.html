Data processing for multiple modes files


Let's understading it on teaching provider level

The provided narrative outlines a strategy for consolidating data from various National Student Survey (NSS) files, which are separated by study modes (e.g., full-time, part-time, apprenticeship), into a unified data model. This integration enables comprehensive analysis and the creation of Key Performance Indicators (KPIs) across different educational dimensions. By appending these datasets into a single fact table, the approach simplifies the analytical process, allowing for a holistic view of student satisfaction and performance without the need to replicate analysis for each study mode separately. This method enhances efficiency and depth of insight derived from the NSS data.

Data manipulation to add mode of study dimension

The datasets we have been provided lack the "Mode of Study" dimension, which means we are unable to directly filter the data by each mode. This dimension is essential for our analysis, particularly when creating DAX calculations, such as benchmarks, and for visualization purposes. While the NSS dashboard includes this dimension, the dataset itself does not, compelling us to create this dimension ourselves for our analysis or to accurately replicate the dashboard published by NSS.
